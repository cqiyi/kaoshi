 数据库原理上机题目连载(一)
几号考数据库原理上机啊？是不是大家都想借这门来得良以上啊？好吧，应爱宝人网友的要求，我这几天会找一些这科的题目以连载的形式贴上来，大家来捧场啊！
以下先引用网友春子的一贴:

 数据库原理上机注意事项
我上星期五刚刚考完，想想下个星期上机的同学，我赶过来给大家发这个帖子。 
题目我觉得万变不离其中,倒没有什么太大的变化.我这次考的是图书管理方面的,无非是建表,插入数据,查询.书本知识一定要掌握，我哪天碰到一个同学，外键约束都不知道怎么回事，那考不过就不能怪题难了。 
这次考试让带书！但是不能带程序进去！我们哪天有人带程序进去，被一个老太太给逮了！！ 
下面我就考试中我碰到的一些细节问题跟大家交流.我在上机前没有用过SQL，所以我也是通过考试的同学了解怎么使用的。 
SQL的界面是这样的： 
从上到下：菜单栏，命令栏（里面你可以输入建表语句，插入语句和查询语句），状态栏，会显示语法检查结果，语句执行结果。 
考试中用到的SQL功能只有两个标准按扭（因为这里不能显示图案，希望下面我的说明大家能明白）： 
一个是勾，也就是对打勾，错打叉的勾，这个是用来判断你的句子语法的； 
再就是一个三角，这个是用来执行你的句子的，这两个按扭挨在一起，在按扭栏的中右一点。当你写完一个句子，必须选中这个句子（所谓一个句子就是一个建表语句，或者查询语句或者插入语句），选中知道什么意思吧（WORD上的选中一样，就是用鼠标左键选中）。然后点击刚才我说的那两个图标，就可以分别进行句子的语法判断和执行了。 

建表和插入中难免会出点小错误，如果你执行后发现了错误，你再修改后，可能会执行不了，为什么呢，它会提示你已经有一个表了，或者不能再插入相同的元组。这时你可以用命令DROP TABLE （表名），把你刚才建的表从库里删除。修改后重新执行，直到得到满意的结果，当然最后要把DROP TABLE （表名）删掉。 
哪天还碰到一个问题，就是书本上的日期类型是DATE，但是SQL7.0里面没有这个类型，只有DATETIME类型。我侥幸碰到的老师（估计是学生）不错，还有我的隔壁的同学的帮忙，总算最后一刻做了出来。下面我把这个类型的用法跟大家说说：把DATE换成DATETIME。日期的输入形式是‘1998-10-30’，注意要有单引号。同样比较的时候是这样的，比如说在1980年1月1日到2050年12月31日：‘1980-1-1’<（变量）<'2050-12-31'，注意也要加上单引号。 

希望大家都能考好！ 
引用完.
 
今天先贴第一题，也就是去年北大系统班考的第二题即北京市考的第一题，原题如下：

某次学校运动会需要保存运动员，比赛项目的信息以及它们之间的关系的信息。运动员有运动员编号，运动员姓名，运动员所在的系。项目有项目编号，项目名字，项目比赛的地点。一个运动员可以参加多个项目，一个项目也可以由多个运动员参加，运动员参加项目按成绩会获得一定的积分。
要求：
1. 建立一个数据库，设计合适的表来存储上述信息，在定义表的时候要给出主外码的信息，并且项目名称和运动员姓名不能为空，积分在0到10之间。
2. 往所建表中插入若干元组，并保证下面所有查询不能为空。
3. 求出计算机系运动员的姓名、所参加的项目名称和取得的成绩；
4. 求出至少参加“王军”同学所参加的一项运动项目的运动员姓名。
5. 统计各系所取得的总成绩，并按积分从高往低排列。
6. 删除总积分低于平均积分的运动员参加项目的信息。
答案如下，已经编译通过：

drop table c--这几句话是为了调试，考试时不要有这几句话。
drop table b--这几句话是为了调试，考试时不要有这几句话。
drop table a--这几句话是为了调试，考试时不要有这几句话。


create table a(
sno char(6),
sname char(8) not null,
sdepartment char(20),
primary key(sname))

create table b(
ino char(6),
iname char(8) not null,
address char(20),
primary key(iname))

create table c(
sname char(8),
iname char(8),
grade smallint,
primary key(sname,iname),
foreign key(sname) references a(sname),
foreign key(iname) references b(iname),
check ((grade between 0 and 10)))

insert into a(sno,sname,sdepartment)

select iname,a.sname,grade
from a,c
where a.sdepartment='计算机' and a.sname=c.sname

select sname
from c
where iname in (select iname
                from c
                where sname='王军' )

select sdepartment,sum(grade)
from a,c
where a.sname=c.sname
group by sdepartment
order by sum(grade)desc
